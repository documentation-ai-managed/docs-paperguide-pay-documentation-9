## Migration Guide

Upgrade smoothly between versions. Follow these steps for the most common transitions.

<Steps>
  <Step title="Update Authentication" icon="shield">

    Replace API keys with JWT tokens.

    <CodeGroup tabs="JavaScript,Python">
    ````javascript
    // Old (v1.0.0)
    const response = await fetch('https://api.example.com/v1/charges', {
      headers: { 'Authorization': `Basic ${btoa('YOUR_API_KEY:')}` }
    });

    // New (v1.1.0+)
    const token = await getJWTToken('your-client-id', 'your-secret');
    const response = await fetch('https://api.example.com/v1/charges', {
      headers: { 'Authorization': `Bearer ${token}` }
    });
    ````

    ````python
    import requests

    # Old (v1.0.0)
    response = requests.post('https://api.example.com/v1/charges',
                             auth=('YOUR_API_KEY', ''))

    # New (v1.1.0+)
    token = get_jwt_token('your-client-id', 'your-secret')
    response = requests.post('https://api.example.com/v1/charges',
                             headers={'Authorization': f'Bearer {token}'})
    ````
    </CodeGroup>

  </Step>
  <Step title="Update Endpoint Paths" icon="link">

    Change from `payment/create` to `charges/create`.

    Test in sandbox: `https://api.example.com/v1/charges?amount=1000&currency=usd`.

  </Step>
  <Step title="Verify Webhooks" icon="zap">

    Re-register webhook endpoints to receive new event types like `payment.refunded`.

  </Step>
</Steps>

<Callout kind="alert">
  Always test migrations in sandbox mode first. Backup your production webhooks before deploying.
</Callout>
